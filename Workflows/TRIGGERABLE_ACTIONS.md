# Workflows — Triggerable Actions Index (Ashfall Britannia)

Every action you can trigger from the `Workflows/` folder and its subfolders: what it does,
and how to fire it. Generated by auditing each file.

**Path shorthand:** `~` = `C:\Users\theli\ashfall_vault` (the vault root). Most Node scripts
hard-code the vault root, but several read files relative to their own folder, so the
**"Run from"** column matters — run them from the listed directory.

---

## Quick reference

| Action | Type | How to trigger |
| --- | --- | --- |
| Process a new session recording | auto + manual | drop an `.mp3` in `~\Session_Sources\Recordings\`, or `node ashfall_pipeline_watch.js` |
| Approve a pending session (apply + push) | manual | double-click `Approve-Ashfall.cmd`, the toast "Approve & apply" button, or `node ashfall_pipeline_watch.js --approve` |
| Open the pipeline status window | manual | double-click `Show-Ashfall-Status.cmd` |
| Install the auto-start watcher task | one-time | double-click `Install-AshfallTask.cmd` (self-elevates) |
| Transcribe an audio file | manual | `node transcribe.js [file]` (in `ashfall_transcribe\`) |
| Sync party character sheets from DDB | auto + manual | `node ddb_party_sync.js` |
| Generate the session-notes .docx | manual | `node ashfall_v1.js` |
| Sync dice rolls DDB → Supabase | extension | "Sync All Campaigns" button in the DDB Roll Sync extension |

## 1. Session pipeline (the core automation)

A new recording is transcribed, spell-checked, paused for your review, then (on approval)
applied to a note, propagated across the vault, and pushed to git.

### `scripts\ashfall_pipeline_watch.js` — watch mode
- **What:** Watches `~\Session_Sources\Recordings\`. When a new `.mp3` finishes copying, it
  refreshes keyterms (if present), **refreshes party sheets** (see §4), transcribes the audio,
  runs Convo 1 Phase A (spell-check only), then **stops and pings you** for review. Applies
  nothing yet. Session number is read from `~\01-Sessions` ("Session NN — …").
- **Trigger (automatic):** drop/record an `.mp3` into the Recordings folder.
- **Trigger (manual):** `node ashfall_pipeline_watch.js`. Leave it running.
- **Run from:** `~\Workflows\scripts`

### `scripts\ashfall_pipeline_watch.js --approve` — approve mode
- **What:** After you've reviewed the spell-check file, runs Convo 1 Phase B (apply
  corrections + generate the session note) then Convo 2 (propagate across the vault + git
  commit/push). Reads the pending session from `~\_pipeline\state.json`.
- **Trigger:** any of —
  - double-click **`scripts\Approve-Ashfall.cmd`**
  - the **"Approve & apply"** button on the review toast (`ashfall_notify.ps1`)
  - the **"Approve & apply"** button in the status window (`ashfall_status_window.ps1`)
  - `node ashfall_pipeline_watch.js --approve`
- **Run from:** `~\Workflows\scripts`

### `scripts\Approve-Ashfall.cmd`
- **What:** Wrapper that calls `ashfall_pipeline_watch.js --approve` in a titled window.
- **Trigger:** double-click, or launched by the toast / status-window approve buttons.

### `scripts\run-watcher.cmd`
- **What:** Launches the watcher and tees output to `~\_pipeline\watcher.log`.
- **Trigger:** double-click, or invoked by `start-watcher-hidden.vbs`.

### `scripts\start-watcher-hidden.vbs`
- **What:** Starts `run-watcher.cmd` with no visible window (stays running so the scheduled
  task can keep/restart the watcher).
- **Trigger:** the "Ashfall Pipeline Watcher" scheduled task at logon (see §7), or double-click
  to start the watcher now without logging out.

### `scripts\Install-AshfallTask.cmd`
- **What:** One-time installer that registers the **"Ashfall Pipeline Watcher"** logon task
  (runs `start-watcher-hidden.vbs` hidden at every logon). Self-elevates via UAC.
- **Trigger:** double-click. **Status: not yet registered** — run this once if you want the
  watcher to auto-start at logon.

## 2. Status window & notifications

### `scripts\Show-Ashfall-Status.cmd` → `scripts\ashfall_status_window.ps1`
- **What:** A small WPF panel showing pipeline progress (Transcription → Spell-check → your
  review → Apply/notes → Propagate+push), driven off `~\_pipeline\status.json` + a tail of
  `watcher.log`. Single-instance; safe to open/close anytime.
- **Triggerable actions inside the window** (visible only during the review gate):
  - **"Open spell-check"** → opens the proposed-corrections file.
  - **"Approve & apply"** → launches `Approve-Ashfall.cmd`.
- **Trigger the window:** double-click `Show-Ashfall-Status.cmd`; the watcher also opens it
  automatically when a run starts.

### `scripts\ashfall_notify.ps1`
- **What:** Shows a Windows toast (BurntToast; falls back to a message box). With
  `-ReviewPath`/`-ApprovePath` it adds **"Review changes"** and **"Approve & apply"** buttons.
- **Trigger:** called by the watcher; not normally run by hand.

## 3. Transcription

### `scripts\ashfall_transcribe\transcribe.js`
- **What:** Transcribes a session recording with AssemblyAI (campaign keyterm boost + custom
  spelling + speaker diarization). Reads `ASSEMBLYAI_API_KEY` from `~\.env` (falls back to the
  sitl_vault `.env`).
- **Trigger:**
  - `node transcribe.js` — interactive picker over the Recordings folder
  - `node transcribe.js "021326 Ashfall Brittania Recording.mp3"`
  - `node transcribe.js --speakers 9 session.mp3` — override speaker count
  - Also called automatically by the watcher.
- **Run from:** `~\Workflows\scripts\ashfall_transcribe`

### `scripts\ashfall_transcribe\ashfall_keyterms_sync.js` *(referenced, not present)*
- The watcher will run this before each transcription **if it exists**. It currently does not,
  so the keyterms-refresh step is a no-op. Add it (mirroring SITL's `sitl_keyterms_sync.js`) to
  enable auto vocabulary growth.

### Ad-hoc transcription helpers (`scripts\ashfall_transcribe\`)
One-off, per-session scripts run manually with `node <file>` as needed — not part of the
standard pipeline:
- `AB_0X_021326_notes.js` … — session-specific notes builders
- `apply_corrections_<date>.js` — apply a correction pass to a transcript
- `merge_022426.js`, `salvage_051126.js` — one-off repair/merge jobs
- `profanity_count.js` — tallies profanity in a transcript

## 4. DDB party character sheets → vault

### `scripts\ddb_party_sync.js`
- **What:** Pulls full character sheets for the party from D&D Beyond and writes raw JSON +
  a readable markdown sheet into `~\03-Characters\01 PCs\Party Character Sheets\`. Output files
  are named after the real character name from each sheet. Anonymous by default; if `DDB_COBALT`
  is set in `~\.env` it authenticates as you and fetches Campaign-Only sheets (the Ashfall party
  sheets all are). Roster lives in `scripts\ddb_party.json`.
- **Trigger:** `node ddb_party_sync.js`. Also called automatically by the watcher each session.
- **Run from:** `~\Workflows\scripts`  ·  **Details + the recovered character IDs:** see
  `scripts\DDB_PARTY_SYNC.md`.

## 5. Session-notes .docx generator

### `scripts\ashfall_v1.js`
- **What:** Generates the styled Ashfall session-notes **.docx** (Ashfall-themed clone of
  SITL's `sitl_v8.js`). After it writes the file, run the documented `fix_tbl_borders`
  post-processing step (strip invalid table borders, repack `word/document.xml`).
- **Trigger:** `node ashfall_v1.js`  ·  **Needs:** the `docx` npm package.
- **Run from:** `~\Workflows\scripts`

## 6. DDB dice rolls → Supabase

### `ddb-roll-sync\` — Chrome extension ("DDB Roll Sync")
- **What:** Captures your DDB Bearer token automatically and syncs new rolls for all active
  campaigns (including **Ashfall Brittania**, `gameId 7170962`) to Supabase, auto-enriching each
  roll (skill/ability/spell/weapon classification, nat20/nat1, dice parsing).
- **Triggerable actions (popup):**
  - **"⚡ Sync All Campaigns"** → fetch + upsert new rolls for every active campaign.
  - **"Clear"** → clear the popup log.
  - **Auto:** token capture runs in the background while you browse DDB.
- **Trigger:** click the extension icon in Chrome, then the button. (Load via
  `chrome://extensions` → Developer mode → Load unpacked → this folder.)

## 7. OS-level trigger

### Scheduled task "Ashfall Pipeline Watcher"
- **What:** Would run `start-watcher-hidden.vbs` at logon so the watcher is always up (hidden).
- **Status:** **not registered.** Create it by double-clicking `Install-AshfallTask.cmd` (§1).
- **Manage (once created):** Task Scheduler → "Ashfall Pipeline Watcher" (Run / End / Disable).
